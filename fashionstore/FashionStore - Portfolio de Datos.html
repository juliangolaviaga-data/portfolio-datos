01 · El desafío

Datos crudos del mundo real

El cliente entregó 3 archivos exportados de su sistema legacy sin ningún estándar de calidad. El objetivo fue construir un pipeline profesional que transforme esos archivos en un modelo limpio, consultable y conectado a dashboards.

📄

clientes_arg.csv — 54 filas

  • 4 clientes duplicados
  • 1 email en mayúsculas
📄

productos_arg.csv — 23 filas

  • 3 filas completamente vacías
  • 14 variantes de 5 categorías (TOPS / Tops / tops)
  • 1 precio inválido (N/A)
  • 9 nombres con espacios sucios
  • 1 stock vacío
📄

ventas_brutas_arg.csv — 300 filas

  • Typo crítico en fecha (2024-06-2O)
  • Formato inválido (21/06-2024)
  • 7 formatos de fecha mezclados

03 · SQL

Staging y exploración

Antes de escribir una sola línea de limpieza, se exploraron las 3 tablas staging con queries estándar reutilizables. Todos los problemas fueron documentados antes de modificar los datos.

Queries de diagnóstico aplicadas

  • Conteo de nulos por columna
  • Detección de duplicados por ID
  • Valores únicos en columnas categóricas
  • Emails con mayúsculas
  • Filas completamente vacías
  • Nombres con espacios al inicio o final
  • Integridad referencial (clientes y productos en ventas)
Ejemplo — Duplicados reutilizable
SELECT columna_id,
       COUNT(*) AS repeticiones
FROM staging_tabla
GROUP BY columna_id
HAVING COUNT(*) > 1;
04 · SQL

Limpieza SQL — Decisiones de negocio

Cada decisión de limpieza tiene una justificación de negocio. No se inventan valores: los nulos son más honestos que un número arbitrario.

Problema Solución SQL Decisión de negocio
Duplicados en clientesDISTINCT ON (cliente_id)Se conserva la primera ocurrencia
Email en mayúsculasLOWER(email)Normalización estándar
Categorías inconsistentesINITCAP(categoria)Estandarización con mayúscula inicial
Nombres con espaciosTRIM(nombre)Limpieza de espacios al inicio y final
Precio N/ACASE WHEN precio = 'N/A' THEN NULLNull es más honesto que inventar un valor
Stock vacíoCASE WHEN stock = '' THEN 0Sin registro de stock se asume 0
Typo en fechaUPDATE SET fecha = '2024-06-20'Corrección quirúrgica del error
Filas vacíasWHERE producto_id IS NOT NULLExclusión de filas sin datos

  ROUND(AVG(ingreso_neto), 2) AS ticket_promedio,
  COUNT(DISTINCT cliente_id)  AS clientes_activos
FROM ventas;

SELECT p.categoria,
       SUM(v.ingreso_neto) AS ingreso_total

08 · SQL Avanzado

CTEs, Window Functions y Subqueries

CTE + NTILE — Segmentación de clientes
WITH ventas_cliente AS (
  SELECT cliente_id,
         SUM(ingreso_neto) AS total
  FROM ventas
  GROUP BY cliente_id
)
SELECT nombre,
  NTILE(3) OVER (
    ORDER BY total DESC
  ) AS tercil
FROM ventas_cliente
JOIN clientes USING (cliente_id);

Clasifica clientes automáticamente en Alto, Medio y Bajo valor.

LAG — Variación mes a mes
ROUND(
  (ventas_mes - LAG(ventas_mes)
    OVER (ORDER BY mes)) * 100.0
  / NULLIF(LAG(ventas_mes)
    OVER (ORDER BY mes), 0),
  2
) AS variacion_mom_pct

Calcula el % de crecimiento o caída respecto al mes anterior.

Subquery — Top producto por ciudad
SELECT ciudad, producto,
       ingreso_total
FROM (...) ranked
WHERE rk = 1
ORDER BY ciudad;
